How It Works

The textbox next to "Enter Text:" is where you input your text that you want either encrypted or decrypted. You then choose what encryption algoritm you want to use. Next you will enter a key (Note not all algorithms use a key see the information about the algorithms below). Then you choose to encrypt or decrypt the text


Monoalphabetic Cipher

The Monoalphabetic Cipher takes a string of capital letters only and numbers or symbols [A-Z0-9!@#] and when encrypted will generate a key that will output to the key textbox and encrypt the letters only [A-Z]. The key that is generated can be used to decrypt the text.


AtBash

AtBash takes any non-special character [a-zA-Z0-9] and inverses swaps it with the character n positions starting from the end of the alphabet and working backwards it is a symmetric cipher that takes no key input.


ROT

The ROT ciphers 1-25 are ceaser ciphers that will shift the characters of the inputted string n times along the alphabet wrapping around. This cipher does not use a key.


Homophonic Substitution

Homophonic Substitution takes the inputted string and switches each character with a predefined dictonary. This cipher does not use a key and can accept any character [a-zA-Z0-9] including special characters.


AES

AES takes a string input including a key and will encrypt the string outputting a digest string. The key is required to decrypt the text.


RC4

RC4 encryption algoritm takes an inputted string and key and returns an encrypted string. The key is requred to decrypt the text.
